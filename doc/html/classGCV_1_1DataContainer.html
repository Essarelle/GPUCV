Detailed Description

Most important class, it is used to manipulate texture properties and data. DataContainer is one of the most important class from GpuCV, it is used to manipulate data of any kind and manage automatique transfer between all possible data locations(Central memory, Graphics memory...).

Member Enumeration Documentation

enum GCV::DataContainer::TextureOptions

Options that can be used with the CL_Options class.

Enumerator:
UBIQUITY 

When true, image can be at several location at the same time, usually when it is an input image. Else only one location is permitted.

CPU_RETURN 

When true, image is transfered back to CPU after a filtered has been applied. Default behavior.

DEST_IMG 

Set the texture as a destination texture, else it is considered as an input texture.

DBG_IMG_TRANSFER 

When true, show debug information on image transfer.

DBG_IMG_FORMAT 

When true, show debug information on image format and properties.

DBG_IMG_LOCATION 

When true, show debug information on image location mechanism.

DBG_IMG_MEMORY 

When true, show debug information on image momery allocation.

PRESERVE_MEMORY 

When true, GpuCV free texture locations when not used(preserve memory but reduce speed).

SMART_TRANSFER 

When true, GpuCV will estimate all transfer time when calling SetLocation() and choose the best one.

VISUAL_DBG 

When true, GpuCV will show all the data container details on image.
