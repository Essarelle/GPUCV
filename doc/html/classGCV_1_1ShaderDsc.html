void GCV::ShaderDsc::ParseMetaTag (  ) 

Parse meta_tags from filename and list them into vectors m_vKeywords and m_vValues. All GpuCV shader files contains some META_DATA that can be updated at compile time. Some of them are set with default values such as {GETTEX, IMGTYPE, GPUCV_FILTER}. It support also customized META_SHADER set by the calling operators, theses arguments are given in the filename. Calling syntax is : shaderfile($keyword1=value1$keyword2=value).frag / .vert The corresponding shader code must have:. 

        #define keyword1 
        #define keyword2

After applying META_TAGS, shader code will be: The corresponding shader code must have: 

        #define keyword1 value1
        #define keyword2 value2
Note:
The '$' character is used to begin the different parameters, cause '$' is not used in GLSL code.
Author:
Yannick Allusse.
Jean-Philippe Farrugia
See also:
ParseShaderCode()
Note:
For geometry shaders: Geometry input/output types must use meta tags GEO_INPUT/GEO_OUTPUT, see m_inputGeometryType/m_outputGeometryType for details.
